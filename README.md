# AI Developer Productivity vs Security Risk

## Overview

This repository explores the trade-off between productivity gains and security risks in AI-assisted software development.

AI coding assistants can help developers complete tasks more quickly, generate code, explain errors, and support troubleshooting. However, faster delivery does not always mean safer delivery. AI-generated outputs may introduce insecure defaults, vulnerable dependencies, weak access controls, misconfigurations, or hidden technical debt.

This project investigates whether the productivity benefits of AI-assisted development are accompanied by measurable increases in security risk.

## Research Question

Do AI-assisted software development workflows improve short-term productivity while increasing long-term security risk?

## Why This Matters

Organisations are rapidly adopting AI tools to accelerate software engineering. However, many adoption decisions focus heavily on productivity and cost savings, while giving less attention to downstream security, remediation, governance, and resilience costs.

This project aims to examine whether AI-assisted development creates a measurable productivity-security trade-off.

## Initial Hypothesis

AI assistance may reduce task completion time, but without strong review and governance, it may also increase the likelihood of insecure code, poor dependency choices, weak configurations, and increased security debt.

## Research Dimensions

This project will explore:

* Developer productivity
* Security vulnerability rates
* Remediation effort
* Code review quality
* Dependency risk
* Infrastructure configuration risk
* Governance and oversight requirements

## Example Metrics

Potential metrics include:

| Metric               | Description                                     |
| -------------------- | ----------------------------------------------- |
| Task completion time | How quickly a development task is completed     |
| Vulnerability count  | Number of security issues identified            |
| Severity score       | Risk level of each vulnerability                |
| Remediation effort   | Estimated time required to fix issues           |
| Review accuracy      | Whether reviewers identify security issues      |
| Dependency risk      | Vulnerability profile of recommended packages   |
| Security debt score  | Combined indicator of future remediation burden |
